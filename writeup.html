Solving the Diet Problem with an Application to a Popular Grocery Chain

The problem

“Adulting,” as we Millenials say, is hard. We finally have to manage our finances, find a non-dissapointing companion, figure out our career, and, most basic – and often most difficult – of all, feed ourselves. You see, we both want food that is simulaneously delicious enough for our weak-willed selves to eat, cheap enough to match our , and nutritous enough to match whatever dietary restrictions we happen to be abiding by at the moment. Luckily, some very smart people have been thinking about this problem for nearly 100 years and have formulated a mathematical approach to solving it.

In the field of convex optimization this problem is known as the the diet problem. It was originally studied in the 1930s and 40s motivated by the Army’s desire to lower costs for supplies while still meeting the nutritional needs of soldiers. The problem goes as follows: given a set of foods, what combination of such foods costs the least and still meet a specified set of nutritional requirements. That is, we have a long menu of food items, each with different costs and nutritional contents, and we want to determine how much of each item we should buy so that we minimize our total cost and meet nutritional our constraints.

c_{1}x_{1} + c_{2}x_{2} + ... + c_{n}x_{n}

Since we are finding an optimal linear combination of foods, we can formulate the problem as a linear program:

minimize c^{T}x \\
subject to Ax \leq b \\
and x \geq 0

With the following variables:

  • $A$, is matrix of $n$ food items, one row for each food; and $m$ column nutritional content of one serving of the food

  • $c$, a vector with $n$ components, each of which representing the cost per serving of each food item

  • $b$, a vector with k components, each representing a the value of a constraint that must be satisfied

  • $x$, a vector with n components, one for each food item, representing the number of servings of each food item in the final menu

A, c, and b are all given. The variable we’re interested in is x, also called the optimization variable. It’s the vector of values for how much of each food we’re going to buy, which we don’t know. We want to find the vector that costs the least and meets all of our constraints. When George Stigler first attempted this problem in the 30s, he found a solution that basically included navy beans and beef liver.

The data

This is all well and good but not yet especially useful. As much as I’d like to, I don’t and wont be making up my diet of meals made from bulk purchases of navy beans and beef liver. I mostly (when not taking out the occasional meal) eat meals assembled out of groceries from Trader Joe’s. Unfortunately, Trader Joe’s doesn’t readily make information about its products avaiable for download. Luckily, they do have have web pages for many items and use a url scheme for those articles that identifies them with (I assume) their index, which happens to just be an integer.

So when we scrape the available fearless flyer items, we end up with 136 products with all the information needed to solve the problem (611 total articles were available), including price, number of servings, calories, protein, fat, carbohydrates, and more. Because many of the diets we’re interested in are expresed in terms of their relative proportions of the different macronutrients, we convert those columns (which are typically expressed in grams of the macronutrient) to calories. This will allow us to specify the proportions of calories we want to come from that macronutrient, rather than specify the how many grams we want.

The code

Now that we have the appropriate data to work with, we can compute an optimal combination of foods. To do so we’ll use CVXPY, “a python-embedded modeling language for convex optimization problems.” 

  # Import the packages
  import cvxpy as cp
  import pandas as pd

  # Read in the data
  df = pd.read_csv('./data/clean/fearless_flyer.csv')

  # Variables
  n = len(df)
  c = df['cost_per_serving'].values
  A = df[['calories', 'fat', 'protein', 'carbs']].values

  # Define the problem and solve
  x = cp.Variable(n, integer=True)
  objective = cp.Minimize(c.T*x)
  constraints = [
      A[:, 0]*x >= min_cals,
      A[:, 0]*x <= max_cals,
      A[:, 1]*x >= min_cals * fat,
      A[:, 1]*x <= max_cals * fat,
      A[:, 2]*x >= min_cals * protein,
      A[:, 2]*x <= max_cals * protein,
      A[:, 3]*x >= min_cals * carbs,
      A[:, 3]*x <= max_cals * carbs,
      x >= 0,
      x <= max_servings
  ]
  problem = cp.Problem(objective, constraints)
  problem.solve()

The cool thing about CVXPY is that problems are specified in code very similarly to how to how they’re specified mathematically. Above we create the variables mentioned above, with c being the vector of costs for each food, A the matrix of foods with their nutrition information, and x, the CVXPY variable to be optimized. To create our objective we just pass the function to be minimized into the Minimize object. The constraints aren’t exactly expressed as the linear inequality $Ax \leq b$, but they’re pretty close and could be. Once we have our objective and constraints we create an instance of the Problem class, which we’ll use to call the solve method.
